SchoolHub - โครงสร้างไฟล์หลังแยกส่วน (จาก SchoolHub_fixed__4_.html)
=======================================================================

ไฟล์เดิม SchoolHub_fixed__4_.html (36,065 บรรทัด / ~1.9MB ไฟล์เดียว)
ถูกแยกออกเป็น:

  index.html   -> โครงหน้าเว็บหลัก (HTML markup ทั้งหมด)
  css/xxx.css  -> ไฟล์ CSS แต่ละก้อน (98 ไฟล์ ตามลำดับเดิมในไฟล์ต้นฉบับ)
  js1/xxx.js   -> ไฟล์ JavaScript ก้อนที่ 001-100
  js2/xxx.js   -> ไฟล์ JavaScript ก้อนที่ 101-124
               (แยกเป็น 2 โฟลเดอร์เพราะมี 124 ไฟล์ เกิน 100 ไฟล์/โฟลเดอร์)

วิธีการแยก (ไม่มีการแก้ไขเนื้อหาโค้ดใดๆ ทั้งสิ้น)
-------------------------------------------------
- ทุก <style>...</style> block ถูกดึงเนื้อหาออกมาเป็นไฟล์ .css แยกไฟล์
  แล้วแทนที่ตำแหน่งเดิมด้วย <link rel="stylesheet" href="css/....css">
  (คง attribute id เดิมไว้ถ้ามี) เรียงลำดับเหมือนต้นฉบับทุกประการ
  (ลำดับ CSS มีผลต่อ specificity)

- ทุก <script>...</script> ที่เป็นโค้ด inline (ไม่ใช่ CDN) ถูกดึงเนื้อหา
  ออกมาเป็นไฟล์ .js แยกไฟล์ แล้วแทนที่ตำแหน่งเดิมด้วย
  <script ... src="js1/....js"></script> หรือ js2 (คง id/type เดิมไว้ครบ
  เช่น type="module" สำหรับสคริปต์ Firebase) เรียงลำดับเดิมทุกประการ

- สคริปต์จาก CDN (TailwindCSS, xlsx-js-style, qrcodejs, tesseract.js)
  ไม่ถูกแตะต้อง ยังเป็น <script src="https://...">  เหมือนเดิม

- ตรวจสอบแล้วว่า: ประกอบไฟล์แยกกลับเป็นไฟล์เดียว ได้เนื้อหาตรงกับ
  SchoolHub_fixed__4_.html แบบ byte-to-byte 100% (ยืนยันด้วยสคริปต์
  automation ไม่ใช่ตรวจด้วยตา)

วิธีใช้งาน
----------
ต้องเปิดผ่านเว็บเซิร์ฟเวอร์ (ห้ามดับเบิลคลิกเปิดแบบ file://)
เพราะมี <script type="module"> (โค้ด Firebase) ซึ่ง browser จะบล็อกการโหลด
ไฟล์ local ผ่าน file:// ตัวอย่าง:

  python3 -m http.server 8000

แล้วเปิด http://localhost:8000
